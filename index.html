Indexing Time Optimization

Ongoing project focused on improving query performance using a bot to suggest which columns should be indexed.

The video above shows how it works: a document (in the form of a table) is chosen and the service returns its suggestions in groups.

In the Symper platform, a service called SYQL will return every SQL query that is ran by any client.

These queries is transferred to the service using Kafka.

The server analyses the query, ask ChatGPT to see which column of the table called in the query should be indexed.

It then returns different combinations of which columns should be indexed (can be single or composite index).

In the future, I want to create a seperate bot that is specifically for suggesting indexes.

The code can be found here: GitHub

Technologies: Python, sql-metadata, ClickHouse, Kafka
